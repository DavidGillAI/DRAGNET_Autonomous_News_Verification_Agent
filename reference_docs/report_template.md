# DRAGNET Report Template

## Purpose

This template defines the structure of the final DRAGNET report.

The report should be clear, factual, transparent and cautious. It should not present speculation, opinion or unsupported claims as fact.

---

# [Story Title]

## Fact-Focused Summary

Write a concise summary of the story using only claims supported by the evidence gathered.

Avoid:

- political framing;
- emotional language;
- speculation;
- unsupported interpretation;
- claims about motive unless directly evidenced.

If the evidence is incomplete, say so clearly.

---

## Timeline, if dates matter

Include this section only when dates matter.

Use bullet points.

- **[Date]**: [Event, statement, correction or update.]
- **[Date]**: [Event, statement, correction or update.]
- **[Date]**: [Event, statement, correction or update.]

Distinguish between:

- when the event happened;
- when a claim was made;
- when an article was published;
- when a correction or update appeared.

## Disputed / Unsupported / Unresolved Claims

Use this section for claims that remain unclear, unsupported or contradicted.

For each claim, explain the status.

### Claim: [Claim]

**Status:** Confirmed / strongly supported / disputed / unsupported / disproven / unresolved

**Assessment:** [Short explanation.]

If no evidence was found, use this wording:

> No supporting evidence was found during this investigation.

Do not automatically describe unsupported claims as false.

---

## Overall Assessment

Explain what the weight of evidence currently supports.

If the evidence is inconclusive, say so clearly.

When appropriate, the agent may offer the simplest interpretation consistent with the available facts, but it must be labelled as an assessment, not established truth.

---

## Confidence Score

**Confidence: [0 to 100]%**

Briefly explain the score.

The score should reflect:

- source quality;
- source independence;
- amount of corroboration;
- access to primary evidence;
- unresolved contradictions;
- retrieval or access limitations explicitly stated in the supplied evidence;
- whether the final summary is fully source-supported.

The percentage is a confidence estimate, not a scientific probability.

---

## Research Transparency

Include only limitations that can be established from the supplied evidence.

State:

- Research iterations completed: [use only the verified runtime count supplied]
- Search coverage: [briefly summarise the supplied research log, including whether follow-up searches added new articles]
- Evidence basis: GNews article metadata, descriptions and content snippets.
- Source limitations: [limitations visible in the supplied evidence]
- Language limitation: English-language sources only.
- Other relevant constraints: [only if supported by the evidence]

Do not invent research iterations, failed tools, paywall status, access status or other actions not explicitly provided.

Each source should include:

1. **[Publication or source name]**
